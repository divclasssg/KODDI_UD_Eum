증거(2026-07-21): 역할극 확인 뒤에만 HTTP question·summary를 호출하고, 검증된 plain text만 세 요약 heading 아래 렌더링한다. provider 질문 실패는 저장 history를 유지한 수동 질문으로, summary 실패·잘못된 evidence는 같은 history의 `fallback-v1-*` 결정론적 요약으로 전환한다. abort·stale 폐기와 raw 오류 비노출을 포함한 관련 단위 테스트 10개 파일·86건, lint, typecheck가 통과했다. actual은 세 합성 Persona 질문 9회·요약 3회까지 통과했지만 전체 UI의 `completed` 저장 여정은 별도 미완료다.

증거(2026-07-22): Persona 없는 공개 `/interview/ai` V2 여정, 질문 안전 validator, 답변 message 근거 요약, 위험 preflight·세 terminal action, exact-token 상태 머신, 원자 저장·reset/abort rollback을 구현했다. 전체 unit 36개 파일·365건, integration 6개 파일·67건, Modal Python 42건, production build와 Chromium E2E 23건, lint·typecheck가 통과했다. deterministic E2E는 실제 AI 성공 증거가 아니며 공개 UI actual 질문 1회·요약 1회는 비용 승인 전이라 미실행 상태다.

증거(2026-07-23): 첫 actual은 오래된 최초 질문 문구 assertion으로 GPU 호출 전에 실패해 harness를 수정했다. 재승인된 두 번째 actual은 실제 질문 GPU 호출 1회가 T4 cold model load 중 85초 web 경계를 넘어 `ClientClosed`·Route HTTP 502로 종료됐고 요약은 호출하지 않았다. 두 시도 모두 자동 반복하지 않았으며 최종 kill switch `1`, 인증 HTTP 503, 실행 container 0을 확인했다. 따라서 Task 1A actual 항목은 계속 미완료다.

추가 증거(2026-07-23): 120초 경계로 실제 질문 `언제부터 불편했나요?`의 공개 UI 표시에는 성공했다. 다만 정적 build에 후속 질문 1회 제한이 빠져 두 번째 GPU 요청도 질문으로 실행됐고 summary Route는 0회였다. build-time 환경 주입을 contract test RED/GREEN으로 수정했으며 unit 3건, typecheck, 관련 lint, diff 검사가 통과했다. 자동 재시도 없이 kill switch `1`, 인증 HTTP 503, container 0을 확인했으므로 실제 요약 gate와 Task 1A는 계속 미완료다.

최종 증거(2026-07-23): 재승인된 공개 UI actual은 합성·비식별 온보딩부터 실제 question 1회·summary 1회 HTTP 200, non-fallback 질문과 AI 근거 요약의 공개 UI 표시를 확인해 Chromium 1건이 1.0분에 통과했다. 종료 뒤 kill switch `1` 재배포, 유효한 인증 V2 probe HTTP 503, 최종 container 0을 확인했다. 따라서 Task 1A actual을 완료 처리한다.

문구 캡처 기록(2026-07-23): 성공 gate 뒤 검증된 UI 문구 출력을 추가하고 별도 재승인으로 실행했으나, cold 모델 로드가 120초를 넘어 question Route HTTP 502와 Modal `ClientClosed`로 실패했다. 질문 GPU 1회, 요약 0회이며 자동 재시도하지 않았다. kill switch `1`, 유효한 인증 V2 probe 503, 최종 container 0을 확인했다. 이는 앞선 Task 1A 성공을 무효화하지 않지만 정확한 요약 문구 캡처는 남아 있다.

180초 문구 캡처 증거(2026-07-23): timeout 계약을 RED/GREEN으로 180초에 맞춘 뒤 공개 actual 1건이 1.4분에 통과했다. 검증된 UI의 질문 영역과 AI 요약 항목은 모두 `합성 상황에서 무릎이 불편해요.`였다. 실제 문구를 확보했지만 질문 영역은 사용자 입력을 반복한 평서문이다. 질문 개수 상한만 검사하고 질문형 자체를 요구하지 않는 validator 결함을 후속 보강 대상으로 기록한다. 종료 뒤 kill switch `1`, 유효한 인증 V2 probe 503, container 0을 확인했다.

validator 보강 증거(2026-07-23): 실제 평서문 출력을 `not-a-question`으로 재현한 RED와 물음표만 붙인 이전 답변 반복을 `repeats-answer`로 재현한 RED를 각각 확인했다. 질문 부호·한국어 의문형·승인된 답변 요청형을 허용하고, 이전 답변과 정규화상 같은 출력은 Modal adapter·Route·browser client·application service에서 거절해 fallback으로 전환한다. 관련 unit 5개 파일·88건, typecheck, lint, diff 검사가 통과했으며 actual은 재실행하지 않았다.
