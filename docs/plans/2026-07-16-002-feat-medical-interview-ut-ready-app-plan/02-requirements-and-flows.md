### Key Flows

- F1. 최초 설치와 기본정보
  - **Trigger:** 처음 앱을 연다.
  - **Steps:** 목적·한계, 로컬 저장, AI 전송 범위를 확인하고 최소 기본정보와 의료정보를 입력한다.
  - **Outcome:** 저장 동의 시 홈에서 새 문진을 시작한다. 저장 거부 시 이유·재검토·종료가 있는 `ConsentBlocked`에 머물고 기록 과업은 사용할 수 없다. AI만 거부하면 홈에서 수동 문진을 시작한다.
  - **Covered by:** R3-R7, R20
- F2. 실제 AI 문진
  - **Trigger:** `새 문진 시작`을 누른다.
  - **Steps:** 입력 방식을 선택하고 주요 증상을 저장한다. MedGemma 질문에 최대 4~5회 답한 뒤 요약 초안을 확인·수정·확정한다.
  - **Outcome:** 확정된 질문·답변과 근거 연결 S/O 요약만 `completed` 기록으로 저장된다. 저장 실패 시 초안은 남고 재시도할 수 있다.
  - **Covered by:** R8-R16, R21
- F3. 의료진에게 오늘 결과 보여주기
  - **Trigger:** 홈에서 오늘 문진을 보여 달라는 요청을 받는다.
  - **Steps:** 기록 메뉴에서 `Asia/Seoul` 기준 오늘의 완료 기록을 선택하고 의료진용 보기를 연다. 완료 기록이 없으면 진행 중 기록과 완료 불가 이유를 구분해 보여 준다.
  - **Outcome:** 불필요한 조작 없이 핵심 요약과 안전 문구를 의료진에게 보여 준다.
  - **Covered by:** R17-R18
- F4. 이전 기록과 내 정보 수정
  - **Trigger:** 과거 기록을 확인하고 기본정보를 바꾸려 한다.
  - **Steps:** 기록 목록에서 이전 날짜를 선택해 상세를 확인하고 내 정보에서 값을 수정한다. 저장 전 취소·이탈에는 변경 폐기 여부를 확인한다.
  - **Outcome:** 저장 성공 뒤에만 현재 profile이 바뀌고 이전 기록 snapshot은 유지된다. 실패 시 편집 초안은 보존된다.
  - **Covered by:** R7, R19
- F5. AI·브라우저 기능 실패
  - **Trigger:** endpoint, quota, timeout, invalid output, 음성 권한, 저장 오류가 발생한다.
  - **Steps:** 현재 입력을 보존하고 오류 이유와 가능한 대체 행동을 한 단계로 제시한다.
  - **Outcome:** 사용자가 문진을 잃지 않고 재시도하거나 수동 경로로 끝낸다. 위험 신호에서는 안내 확인 뒤 안전 종료 기록 또는 허용된 수동 요약으로만 이동한다.
  - **Covered by:** R9-R10, R15-R16
- F6. 데이터 reset
  - **Trigger:** 공용 기기에서 새 사용자를 시작하거나 사용자가 데이터 삭제를 요청한다.
  - **Steps:** 삭제 범위를 확인하고 TTS·음성 입력을 중단한 뒤 IndexedDB를 지운다.
  - **Outcome:** 이전 의료정보가 다음 사용자에게 노출되지 않는다.
  - **Covered by:** R20

### Acceptance Examples

- AE1. A1 조건에서 화면은 18px 이상의 짧은 문장과 명시적 라벨을 사용하고 질문을 TTS로 들을 수 있다.
- AE2. A2 조건에서 `숨쉬기 힘들어요?`처럼 구체적인 질문 하나와 `예/아니오/모르겠음`만 먼저 제시한다.
- AE3. A3 조건에서 칩에서 음성으로 바꾸어도 이미 선택한 증상과 초안이 유지된다.
- AE4. 실제 MedGemma가 두 질문, 전문용어, 진단·치료 지시 또는 이미 답한 질문을 반환하면 표시하지 않는다.
- AE5. S/O 항목의 근거 ID가 없거나 수치·날짜·단위가 원문과 다르면 저장하지 않고 수동 요약으로 전환한다.
- AE6. 오늘 기록과 과거 기록이 함께 있을 때 오늘 기록이 날짜와 `오늘` 라벨로 식별된다.
- AE7. 의료진용 보기에서는 편집·내비게이션보다 요약, 원문 보기, 안전 문구, 닫기만 우선 노출된다.
- AE8. 기본정보를 수정해도 이미 완료한 기록의 profile snapshot은 바뀌지 않는다.
- AE9. 음성 권한을 거부해도 텍스트 입력으로 즉시 돌아가며 입력 초안이 남는다.
- AE10. TTS 재생 중 화면 이동·숨김·reset이 발생하면 발화가 중단된다.
- AE11. 사진 처리 전체가 검증되지 않은 build에서는 사진 버튼을 비활성 placeholder로 보여 주지 않고 숨긴다.
- AE12. 데이터 전체 삭제를 확인하면 profile, interview, attachment, summary가 모두 제거된다.
- AE13. 요약 확인 전에는 기록 상태가 `draft`이며 의료진 보기를 열 수 없다. 저장 실패 뒤 수정본을 유지한 채 재시도할 수 있다.
- AE14. 위험 신호 안내를 확인한 뒤 안전 종료 상태가 저장되고, 뒤로가기·새로고침·늦은 AI 응답으로 일반 문진이 다시 시작되지 않는다.
- AE15. profile 저장 실패·취소·이탈 시 기존 profile과 과거 snapshot은 바뀌지 않고, 저장 실패 시 편집 초안만 남는다.

### Three-Persona × Three-Task Acceptance Matrix

| Persona | Task 1 — 최초 문진 | Task 2 — 오늘 기록 보여주기 | Task 3 — 과거 기록·내 정보 수정 |
|---|---|---|---|
| 김영수 | onboarding의 목적·동의를 쉬운 단문·한 단계·명시적 라벨과 TTS로 확인한 뒤 큰 글씨로 실제 질문과 요약 확정 | `오늘`·완료 상태·시간·주요 증상을 큰 라벨로 구분하고 한 단계씩 clinician view 진입 | 현재 값을 먼저 보여 주고 짧은 섹션 단위로 수정, 취소·저장 결과를 쉬운 문장으로 안내 |
| 이민정 | onboarding부터 한 화면 한 행동·복문 없는 구체 설명·이전 단계 복귀·입력 보존을 적용하고, 한 질문과 구체 선택지로 완주 | 기록 선택과 보여주기를 한 화면 한 단계로 진행하며 추상 아이콘에 의존하지 않음 | 일관된 내비게이션과 짧은 필드 설명으로 수정, 오류는 해당 필드에서 구체적으로 안내 |
| 박성훈 | 칩·짧은 답변·음성 입력을 오갈 때 draft 유지 | 긴 입력 없이 최신 완료 기록을 찾고 큰 `의료진에게 보여주기` 버튼 사용 | 반복 입력 없이 기존 값을 수정하고 keyboard·짧은 입력만으로 저장 |
