## 핵심 결정

### 구현 순서

Next.js 설치와 최소 폴더를 먼저 만들고 Figma Variables를 CSS 디자인 토큰으로 옮긴다. 대표 화면은 `/interview/new`로 정한다. 다만 대표 UI가 완성될 때까지 개발 전체를 기다리지 않고 타입, 상태, IndexedDB, API 경계를 병렬로 만든다.

### 일정

- 6일차: 핵심 MVP와 데모 흐름 동결
- 7~9일차: 음성, 이미지, PIN, PWA 캐시, 접근성, 회귀 테스트
- 9일차: 신규 기능을 추가하지 않고 안정화와 리허설만 수행

### TTS

PWA에서는 사용자가 누르는 읽기 버튼을 기본으로 하며 자동 재생은 끈다. 음성 기능이 실패하더라도 모든 정보와 조작이 텍스트로 유지되어야 한다. iOS 네이티브 구현은 별도 앱 개발이므로 현재 프로토타입 범위에서 제외한다.
