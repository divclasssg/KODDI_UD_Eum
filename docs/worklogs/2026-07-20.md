## 결정

- 외부 사용자가 링크를 아는 경우 로그인 없이 접근하는 데모로 정했다.
- 실제 환자 정보 대신 합성 Persona 역할극만 허용한다.
- 자유 텍스트는 허용하되 실제 정보 오입력을 완전히 차단할 수 없다는 잔여 위험을 기록했다.
- MedGemma 실행 기반을 Vertex AI에서 Modal custom App으로 변경했다.
- 브라우저는 Modal을 직접 호출하지 않고 Next.js Route Handler만 인증 endpoint를 호출한다.
- T4 1개, scale-to-zero, 최대 GPU container 1개, 60초 deadline으로 시작한다.
- session 5회/분, IP 20회/시간, 전체 actual 100회/일과 server kill switch를 둔다.
- Modal Workspace budget은 월 $30 hard cap으로 설정한다.
- 음성 입력은 실제 마이크·녹음·STT 없이 합성 Persona transcript를 채우는 모의 기능으로 정했다.
- 질문 수가 상황에 따라 달라지므로 단계 번호와 고정 진행률을 표시하지 않는다.

- 전화번호·이메일·주민등록번호·주소/기관 표지만 반환하는 직접 식별정보 탐지기를 추가했다.
- 공유 command port에 저장·다음 질문·요약·안전 행동을 모으고 기존 fixture 동작과 호출 횟수를 유지했다.
- 식별정보 탐지기가 실제 건강정보 판별기가 아니며 오탐·미탐 가능성이 있음을 한글 주석으로 기록했다.

## Modal Task 2

- server-only provider factory와 결정론적 mock·Modal adapter를 구현했다.
- Modal proxy 인증 header와 `kind`, context, session/IP HMAC만 포함하는 body allowlist를 고정했다.
- 60초 timeout, `429`·`503`·일시적 transport 오류 1회 재시도, 인증·schema 오류 무재시도를 구현했다.
- Origin·JSON·8,192 byte·직접 식별정보를 provider 호출 전에 검사하는 공통 Route Handler를 추가했다.
- 익명 UUID의 HttpOnly cookie와 session/IP HMAC-SHA256을 적용하고 실제 원문은 Modal quota 식별자에 포함하지 않는다.
- Node.js question·summary Route Handler와 `mock|modal` 환경 계약을 추가했다.

## Modal Task 3

- Python 3.12 Modal image와 고정된 runtime package 계약을 코드로 추가했다.
- Pydantic exact schema와 UTC minute·hour·day quota bucket을 구현했다.
- session 5회/분, IP 20회/시간, actual 100회/일을 초과하면 GPU 호출 전에 거절한다.
- prompt에서 진단·치료·복약 지시를 금지하고 untrusted context를 구조적으로 구분했다.
- 모델을 image build 단계에서 내려받고 T4 class 진입 시 float16으로 한 번 로드하도록 구성했다.
- CPU gate를 container 1개·동시 입력 1개로 직렬화하고 인증 endpoint만 노출했다.
- GPU·모델 Secret이 없는 별도 quota smoke app을 추가했다.
- Pydantic 기본 422 대신 승인된 400을 보장하기 위해 endpoint 내부에서 schema를 직접 검증한다.
