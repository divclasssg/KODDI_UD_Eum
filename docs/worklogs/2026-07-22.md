## U2 IndexedDB v1 구현

- `koddi-ud-eum` version 1에 consent, profile, medical profile, interview, draft, message, summary, attachment 8개 store와 index를 고정했다.
- timestamp는 UTC ISO 8601 millisecond `Z`, 문진 status는 `draft | review | completed | safety-stopped`, revision은 1부터 transaction마다 증가하도록 했다.
- local 동의가 없는 profile·interview read/write를 차단하고 AI 거부 상태에서는 manual 로컬 저장만 허용하는 policy를 분리했다.
- draft·commit history·summary를 한 readonly snapshot으로 복원하고 revision·message sequence·summary evidence 불일치를 거절한다.
- 완료 transaction에서 현재 profile과 medical profile을 deep snapshot으로 저장하고 terminal record 재완료를 막았다.
- reset은 8개 store를 한 readwrite transaction에서 지우며, 강제 abort 시 전량 보존되고 성공 뒤 모두 0건임을 확인했다.
- runtime generation과 transaction 내부 consent/interview/revision 검사로 reset 뒤 늦은 응답의 데이터 부활을 차단했다.
- migration 실패 시 기존 data와 version을 보존하고 blocked·higher version·versionchange를 typed 정책으로 처리했다.

## U3 문진 상태·입력 계약 완료

- 공개 `/interview/manual`을 `UI → application service → pure domain machine → repository port`로 재구성했다. pure domain은 React, router, IndexedDB를 import하지 않는다.
- `interviewId`는 durable identity, `revision`은 성공한 aggregate transaction의 concurrency token, `requestId`는 runtime effect identity, runtime generation은 reset epoch로 분리했다.
- exact operation token의 session·request·interview·base revision·generation 중 하나라도 다르면 늦은 성공과 실패를 모두 UI state에서 폐기한다. 저장소는 revision과 runtime generation을 별도로 검사한다.
- IndexedDB database version 1과 기존 8개 store/index를 유지하면서 신규 interview·draft V2에 immutable question-set snapshot과 mode별 common draft를 저장한다. 진행 중 V1 draft는 load 시 한 transaction으로 명시적으로 V2 승격한다.
- text·choice·chip·measurement adapter와 pure validation을 구현했다. 기간·강도는 기존 승인 문구로 공개 chip에 적용했고, 공개 증상 preset과 공개 measurement 질문은 의료 콘텐츠 승인 전까지 추가하지 않았다.
- measurement는 raw decimal, unit ID, local 측정 시각과 `empty | known | unknown`을 분리하며 unknown 왕복과 mode 전환에도 직전 known 값을 보존한다. 제출 가능한 known answer는 local 값을 UTC ISO `measuredAt`으로 직렬화한다.
- 저장 중 submit은 최신 draft persist 뒤 새 revision으로 한 번만 실행된다. navigation과 unload를 차단하고 저장 실패 뒤 입력을 보존한 채 재시도를 허용한다.
- 실제 IndexedDB integration과 Chromium에서 text↔chip 전환 값을 reload 뒤 복원했고 AI·media·STT request가 0건임을 확인했다.
