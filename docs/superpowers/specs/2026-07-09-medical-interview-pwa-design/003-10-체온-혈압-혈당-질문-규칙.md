## 10. 체온, 혈압, 혈당 질문 규칙

체온, 혈압, 혈당은 항상 묻지 않는다.

이 값들은 현재 증상에 도움이 될 때만 물어본다.

예:

- 열이 난다고 하면 체온을 물어볼 수 있다.
- 어지럽다고 하면 혈압이나 혈당을 물어볼 수 있다.
- 숨이 차다고 하면 산소포화도 같은 값이 있는지 물어볼 수 있다.

하지만 사용자가 모를 수 있다.

그래서 반드시 아래 선택지를 제공한다.

- 입력하기
- 모름
- 건너뛰기

답하지 않아도 문진은 계속 진행된다.  
답하지 않아도 S/O 요약은 만들 수 있다.

입력한 체온, 혈압, 혈당은 그날 문진 기록에만 저장한다.  
기본 의료정보에는 저장하지 않는다.

## 11. 저장되지 않은 의료정보를 물어보는 방식

문진 중 필요한 정보가 기본 의료정보에 없을 수 있다.

예:

- 복용약이 저장되어 있지 않다.
- 알레르기가 저장되어 있지 않다.
- 가족력이 저장되어 있지 않다.
- 기존 질환이 저장되어 있지 않다.

이때 AI는 질문할 수 있다.

예:

```text
복용 중인 약이 있나요?
저장된 정보가 없어서 한 번만 확인할게요.
```

사용자가 답하면 앱은 이렇게 물어본다.

```text
이 정보를 기본 의료정보에 저장할까요?
[저장하기] [이번 문진에만 사용]
```

AI가 직접 저장하지 않는다.  
앱이 사용자에게 확인한다.  
사용자가 허락해야 저장한다.

저장 후보 예:

```json
{
  "nextQuestion": "복용 중인 약이 있나요?",
  "saveCandidate": {
    "category": "medications",
    "value": "혈압약",
    "savePolicy": "confirm",
    "reason": "다음 문진에서 다시 묻지 않기 위해"
  }
}
```

## 12. Hugging Face와 MedGemma 연결

사용할 모델:

```text
google/medgemma-1.5-4b-it
```

이 모델을 쓰는 이유:

- 4B 모델이라 비교적 가볍다.
- 글과 이미지를 함께 받을 수 있다.
- MedGemma 1.5 계열이다.
- 무료 범위 데모에 가장 현실적이다.

연결 방식:

1. 사용자가 앱에서 문진을 입력한다.
2. Next.js 서버 API가 요청을 받는다.
3. 서버 API가 Hugging Face에 요청을 보낸다.
4. Hugging Face가 MedGemma 결과를 돌려준다.
5. 앱이 결과를 화면에 보여준다.

중요한 보안 규칙:

- Hugging Face 토큰은 브라우저에 넣지 않는다.
- 토큰은 서버의 `.env` 파일에만 둔다.
- 사용자의 이름과 연락처는 AI에 보내지 않는다.

무료 범위에서만 데모한다.

무료 사용량이 끝나거나 연결이 실패하면 이렇게 보여준다.

```text
AI 응답을 잠시 사용할 수 없어요.
잠시 후 다시 시도해 주세요.
```

사용자는 다시 시도할 수 있어야 한다.  
AI가 안 되어도 수동 요약 화면을 볼 수 있어야 한다.

## 13. 사진과 영상 처리

### 사진

- 사용자는 사진을 첨부할 수 있다.
- 사진은 필요할 때 MedGemma에 보낼 수 있다.
- AI는 사진을 보고 병명을 말하면 안 된다.
- AI는 사진에 대해 확실하지 않은 해석을 하면 안 된다.
- 사진은 의사가 직접 확인할 수 있도록 첨부로 보여준다.

### 영상

- 영상은 분석하지 않는다.
- 영상은 첨부 파일로만 저장한다.
- 요약에는 파일명과 첨부 여부만 보여준다.
- 의사가 필요하면 직접 확인한다.

## 14. 음성 입력

브라우저 음성 인식을 우선 사용한다.

필요한 기능:

- 말하기 시작 상태 표시
- 말하기 종료 상태 표시
- 인식된 문장을 사용자가 고칠 수 있음
- 음성 인식 실패 시 글 입력으로 바꿀 수 있음
- 사용자가 확인하기 전에는 자동 제출하지 않음

## 15. 저장 방식

사용자 기기 안에 저장한다.

저장 위치:

- 기본 저장: IndexedDB
- 간단한 설정: localStorage 사용 가능

저장하는 것:

- 기본 의료정보
- 문진 기록
- 첨부 파일 정보
- AI 사용 동의 여부
- 잠금 설정
- 저장 후보 정보
