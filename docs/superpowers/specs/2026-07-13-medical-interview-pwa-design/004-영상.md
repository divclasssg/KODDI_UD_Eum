- 의사가 필요하면 직접 확인한다.

## 14. 음성 입력

브라우저 음성 인식을 우선 사용한다.

필요한 기능:

- 말하기 시작 상태 표시
- 앱이 듣고 있다는 것을 보여주는 오디오 웨이브 또는 비슷한 움직임
- 말하기 종료 상태 표시
- 인식된 문장을 사용자가 고칠 수 있음
- 음성 인식 실패 시 글 입력으로 바꿀 수 있음
- 사용자가 확인하기 전에는 자동 제출하지 않음

말하기 종료 규칙:

- 사용자가 말하다가 잠시 멈출 수 있다.
- 그래서 너무 빨리 말하기를 끝내면 안 된다.
- 음성 인식 중 3.5초 이상 말소리가 감지되지 않으면 말하기가 끝난 것으로 본다.
- 그 뒤 텍스트 변환 단계로 넘어간다.
- 사용자는 변환된 문장을 확인하고 고칠 수 있어야 한다.

## 15. 저장 방식

별도 데이터베이스는 만들지 않는다.

사용자 기기 안에 저장한다.

저장 위치:

- 기본 저장: IndexedDB
- 간단한 설정: localStorage 사용 가능

저장 처리 규칙:

- 프로필, 의료정보, 문진 답변, 첨부 파일은 IndexedDB 트랜잭션이 완료된 뒤 저장 성공으로 처리한다.
- 저장 중에는 `다음`, `저장`, 선택 버튼을 비활성화하고 로딩 상태를 글자로 보여준다.
- 같은 동작을 빠르게 여러 번 눌러도 레코드를 한 번만 저장한다.
- 저장 실패 시 화면을 이동하지 않고 사용자의 입력을 유지한다.

PWA 업데이트 규칙:

- 서비스 워커의 Cache Storage와 사용자 데이터가 있는 IndexedDB를 분리한다.
- 서비스 워커 설치, 활성화, 오래된 캐시 삭제 과정에서 IndexedDB를 삭제하지 않는다.
- IndexedDB 스키마 변경은 기존 레코드를 보존하는 버전 마이그레이션으로 처리한다.
- 브라우저 데이터 삭제까지 막을 수는 없으므로 영구 보존을 약속하지 않는다.

사진 저장 규칙:

- 사진은 IndexedDB에 저장하기 전에 반드시 압축한다.
- 사진의 긴 쪽 길이는 최대 1024px 정도로 줄인다.
- 압축된 사진만 저장한다.
- 원본 사진을 그대로 저장하지 않는다.
- 저장 공간이 부족하면 사용자에게 알려준다.

저장하는 것:

- 기본 프로필
- 의료정보
- 문진 기록
- 첨부 파일 정보
- AI 사용 동의 여부
- 잠금 설정
- 저장 후보 정보

기본 프로필에 저장하는 것:

- 생년월일 또는 나이
- 성별

의료정보에 저장하는 것:

- 키
- 몸무게
- 기존 질환
- 복용약
- 알레르기
- 가족력
- 수술/시술 이력
- 흡연/음주 여부

문진 기록에만 저장하는 것:

- 오늘 증상
- 통증 강도
- 시작 시점
- 증상 변화
- 오늘 잰 체온
- 오늘 잰 혈압
- 오늘 잰 혈당
- 사진 첨부
- 영상 첨부

삭제 기능:

- 모든 기록 삭제
- 기본 프로필 삭제
- 의료정보 삭제
- 특정 문진 삭제

## 16. 개인정보와 동의

이름, 연락처, 주민등록번호 같은 정보는 AI에 보내지 않는다.

AI에 보낼 수 있는 정보:

- 나이
- 성별
- 키/몸무게
- 기존 질환
- 복용약
- 알레르기
- 가족력
- 현재 증상
- 첨부 사진

처음 사용할 때 아래 안내를 보여준다.

```text
AI 요약 생성을 위해 입력한 증상 정보가 외부 AI 서비스로 전송될 수 있어요.
이름이나 연락처 같은 정보는 보내지 않아요.
```

## 17. 응급 신호 처리

앱은 응급인지 진단하지 않는다.

하지만 위험할 수 있는 말이 입력되면 안전 안내를 보여준다.

예:

- 가슴 통증과 숨참
- 갑자기 한쪽 팔이나 다리에 힘이 빠짐
- 의식이 흐림
- 심한 호흡곤란
- 갑자기 심한 두통
- 피가 많이 남

안내 예:

```text
응급일 수 있어요.
증상이 심하거나 계속되면 즉시 주변 사람이나 의료진에게 알려주세요.
```

이 안내는 진단이 아니다.  
안내가 떠도 문진과 요약 생성은 계속할 수 있다.
