## 9. AI 질문 규칙

질문은 기본적으로 4~5개 안에서 끝낸다.

필요한 정보가 충분하면 4개보다 적어도 요약 단계로 이동한다.

이미 이번 문진에서 답한 정보는 다시 묻지 않는다.

기본 질문 예:

1. 가장 불편한 증상은 무엇인가요?
2. 언제부터 시작됐나요?
3. 어디가, 어느 정도로 불편한가요?
4. 좋아지거나 나빠지는 상황이 있나요?
5. 복용약, 기존 질환, 알레르기 중 알려야 할 것이 있나요?

저장된 의료정보가 있으면 기본값으로 참고한다.

하지만 아래 경우에는 다시 확인할 수 있다.

- 현재 증상과 관련된 복용약, 알레르기, 기존 질환을 이번 문진에서 아직 확인하지 않았다.
- 몸무게, 흡연/음주 여부처럼 바뀔 수 있는 정보가 마지막 확인 후 6개월이 지났고 현재 문진에 필요하다.
- 변화 과정이 진료에 중요할 수 있다.

예를 들어 복용약, 알레르기, 몸무게, 흡연/음주 여부는 바뀔 수 있다.  
이런 정보는 “저장된 복용약 정보가 지금도 맞나요?”처럼 확인할 수 있다.

각 의료정보에는 마지막 확인 시각을 저장한다.

6개월이 지났다는 이유만으로 현재 문진과 관련 없는 정보를 무조건 묻지 않는다.

사용자가 `모르겠음`을 선택하면 기존 저장값을 최신 정보로 덮어쓰지 않고 요약의 확인 필요 항목에 표시한다.

기존 질환이나 가족력처럼 잘 바뀌지 않는 정보는 보통 다시 묻지 않는다.  
다만 현재 증상과 관련이 크면 확인할 수 있다.

## 10. 체온, 혈압, 혈당 질문 규칙

체온, 혈압, 혈당은 항상 묻지 않는다.

이 값들은 현재 증상에 도움이 될 때만 물어본다.

예:

- 열이 난다고 하면 체온을 물어볼 수 있다.
- 어지럽다고 하면 혈압이나 혈당을 물어볼 수 있다.
- 숨이 차다고 하면 산소포화도 같은 값이 있는지 물어볼 수 있다.

하지만 사용자가 모를 수 있다.

그래서 바로 값을 요구하지 않는다. 먼저 측정값이 있는지 묻는다.

선택지는 Figma 기준에 맞춘다.

- 예
- 아니오
- 모르겠음

사용자가 `예`를 선택하면 값을 입력한다.  
사용자가 `아니오` 또는 `모르겠음`을 선택하면 값을 입력하지 않고 다음 단계로 간다.

선택 버튼을 누르는 행위는 사용자의 확정 동작이다.

버튼을 누르면 선택값을 바로 저장하고, 처리 중에는 선택 버튼을 비활성화한다.

자유 텍스트와 음성 인식 결과는 사용자가 내용을 확인한 뒤 제출한다.

답하지 않아도 문진은 계속 진행된다.  
답하지 않아도 S/O 요약은 만들 수 있다.

입력한 체온, 혈압, 혈당은 그날 문진 기록에만 저장한다.  
기본 프로필이나 의료정보에는 저장하지 않는다.

## 11. 저장되지 않은 의료정보를 물어보는 방식

문진 중 필요한 정보가 기본 프로필이나 의료정보에 없을 수 있다.

예:

- 복용약이 저장되어 있지 않다.
- 알레르기가 저장되어 있지 않다.
- 가족력이 저장되어 있지 않다.
- 기존 질환이 저장되어 있지 않다.

이때 AI는 질문할 수 있다.

예:

```text
복용 중인 약이 있나요?
저장된 정보가 없어서 한 번만 확인할게요.
```

사용자가 답하면 앱은 이렇게 물어본다.

```text
이 정보를 의료정보에 저장할까요?
[저장하기] [이번 문진에만 사용]
```

AI가 직접 저장하지 않는다.  
앱이 사용자에게 확인한다.  
사용자가 허락해야 저장한다.

저장 후보 예:

```json
{
  "nextQuestion": "복용 중인 약이 있나요?",
  "saveCandidate": {
    "category": "medications",
    "value": "혈압약",
    "savePolicy": "confirm",
    "reason": "다음 문진에서 다시 묻지 않기 위해"
  }
}
```

## 12. Hugging Face와 MedGemma 연결

사용할 모델:

```text
google/medgemma-1.5-4b-it
```

이 모델을 쓰는 이유:

- 4B 모델이라 비교적 가볍다.
- 글과 이미지를 함께 받을 수 있다.
- MedGemma 1.5 계열이다.
- 무료 범위 데모에 가장 현실적이다.

연결 방식:

1. 사용자가 앱에서 문진을 입력한다.
2. Next.js 서버 API가 요청을 받는다.
3. 서버 API가 Hugging Face에 요청을 보낸다.
4. Hugging Face가 MedGemma 결과를 돌려준다.
5. 앱이 결과를 화면에 보여준다.

중요한 보안 규칙:

- Hugging Face 토큰은 브라우저에 넣지 않는다.
- 토큰은 서버의 `.env` 파일에만 둔다.
- 사용자의 이름과 연락처는 AI에 보내지 않는다.

무료 범위에서만 데모한다.

무료 사용량이 끝나거나 연결이 실패하면 이렇게 보여준다.

```text
AI 응답을 잠시 사용할 수 없어요.
잠시 후 다시 시도해 주세요.
```

사용자는 다시 시도할 수 있어야 한다.  
AI가 안 되어도 수동 요약 화면을 볼 수 있어야 한다.
