## 19. 오류 처리

### AI 호출 실패

원인:

- 무료 사용량 초과
- 네트워크 오류
- 모델 로딩 실패
- 너무 많은 요청

처리:

- 다시 시도 버튼 제공
- 잠시 후 다시 시도 안내
- 수동 요약 보기 제공

### 음성 인식 실패

처리:

- 글 입력으로 바꿀 수 있게 한다.

### 첨부 실패

처리:

- 파일 크기나 파일 형식을 알려준다.
- 다시 첨부할 수 있게 한다.

### 저장 실패

처리:

- 저장 공간이 부족할 수 있음을 알려준다.
- 삭제 또는 다시 시도 방법을 알려준다.

## 20. 테스트 기준

### 기능 테스트

- Figma 기준 주요 화면이 있다.
- 아래쪽 탭, 헤더, 버튼, 입력창이 Figma와 비슷하다.
- 기본 프로필을 입력하면 저장된다.
- 키와 몸무게는 의료정보로 저장된다.
- 저장된 정보는 다음 문진에서 다시 사용된다.
- 글 입력으로 문진을 끝낼 수 있다.
- `예`, `아니오`, `모르겠음`은 한 번 누르면 글 입력창을 거치지 않고 저장된다.
- 선택 답변 저장 중 중복 클릭으로 같은 답변이 두 번 생기지 않는다.
- 저장 완료 전에 다음 화면으로 이동하지 않는다.
- 필요한 정보가 충분하면 4개보다 적어도 요약으로 넘어간다.
- 직전 답변 수정 시 이후 질문과 임시 요약을 다시 계산한다.
- 음성 입력 결과를 고칠 수 있다.
- 음성 입력 중 앱이 듣고 있다는 표시가 보인다.
- 3.5초 이상 말소리가 없으면 말하기 종료로 처리된다.
- 사진을 첨부할 수 있다.
- 사진은 저장 전에 최대 1024px 정도로 압축된다.
- 영상은 첨부 목록에만 보인다.
- AI 실패 시 다시 시도할 수 있다.
- AI 실패 시 수동 요약을 볼 수 있다.
- 의료진용 S/O 요약이 만들어진다.
- 서비스 워커를 업데이트해도 IndexedDB의 프로필과 문진 기록이 유지된다.
- PIN 잠금 후 새로고침하거나 브라우저를 다시 열면 잠금 화면이 먼저 나온다.
- PIN 확인 전에는 프로필, 문진 기록, 요약이 잠깐이라도 보이지 않는다.

### 안전 테스트

- AI가 진단명을 만들지 않는다.
- AI가 치료 방법을 추천하지 않는다.
- AI가 약 복용을 지시하지 않는다.
- 응급 신호는 진단이 아니라 안전 안내만 보여준다.
- 이름과 연락처는 AI 요청에 포함되지 않는다.

### 접근성 테스트

- 모바일 화면에서 버튼 크기가 최소 48px 이상이다.
- 본문 글자 크기가 최소 16px 이상이다.
- 글자를 키워도 내용이 잘리지 않는다.
- 색깔 없이도 상태를 이해할 수 있다.
- 화면 읽기 순서가 자연스럽다.

### 디자인 비교 테스트

- Figma `최종 prototype`과 실제 화면을 비교한다.
- 393px 모바일 폭에서 Figma와 비슷하게 보인다.
- `Buttons`, `Chat_Input`, `speech_bubble`, `Summary_Item`, `FormInput`, `GNB`, `Header` 역할을 따른다.
- Figma에 없는 설명 화면을 마음대로 많이 추가하지 않는다.
