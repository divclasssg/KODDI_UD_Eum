# 최종 프로토타입 필수 UI 목록

## 목적

Figma `prototype` 페이지의 `최종 prototype` 섹션을 제품 화면의 시각 기준으로 고정하고, 구현해야 할 UI와 대응 node를 누락 없이 식별한다. 이 문서는 화면 범위만 확정하며 컴포넌트·토큰 매핑과 상세 상태 계약은 후속 명세에서 다룬다.

## 원본과 판정 규칙

- 기준 파일: `Prototype` (`8HVgkA1i3ujI5Eg2kNSI68`)
- 기준 페이지: `prototype` (`94:2275`)
- 기준 섹션: `최종 prototype` (`538:5661`)
- 섹션의 직접 자식인 제품 `FRAME` 46개를 필수 시각 참조로 사용한다.
- 같은 라우트의 입력값·키보드·대기·선택·첨부 상태는 별도 화면이 아니라 상태 변형으로 구현한다.
- 중첩 프레임은 직접 자식 화면의 내부 구조이므로 별도 필수 화면 수에 더하지 않는다.
- 설명용 `Frame 228` (`324:5924`), 주석 텍스트, 참조 이미지와 섹션 밖 과거 시안은 제외한다.
- Figma에 없는 오류·안전·저장·AI 대기 상태는 제품 요구사항에 따라 확장하되 Figma 원본 화면으로 오인하지 않는다.

## 구현 시 보존할 공통 UI

- 393px 모바일 앱 셸과 상단 상태·뒤로가기 영역
- 홈·기록·프로필 전역 내비게이션
- 한 번에 하나의 질문, 이전 답변 요약, 현재 입력, 주요 다음 행동
- 텍스트·음성·선택형 입력 전환과 입력값 보존
- 기록 카드, 환자용 상세, 의료진용 요약의 정보 위계
- 빈 상태와 데이터 상태, 취소·저장·추가·종료 행동
- 조건부 사진 기능을 노출할 때의 추가·미리보기·완료 상태
