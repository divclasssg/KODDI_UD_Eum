# 의료 취약군 문진 보조 앱 설계서

작성일: 2026-07-10

## 1. 만들려는 것

병원에서 증상을 말하기 어려운 사람이 있다.

예를 들면:

- 말로 길게 설명하기 어려운 사람
- 글을 읽거나 쓰기 어려운 사람
- 병원에서 긴장해서 증상을 잘 말하지 못하는 사람
- 고령자
- 발달장애인
- 실어증 등으로 말하기가 어려운 사람

이 앱은 이런 사람이 병원에 가기 전에 증상을 쉽게 기록하도록 돕는다.

사용자는 글, 음성, 사진, 선택 버튼으로 증상을 입력한다.  
AI는 필요한 내용을 짧게 물어본다.  
마지막에는 의사가 보기 쉬운 요약을 만든다.

이 앱은 병을 진단하지 않는다.  
이 앱은 치료 방법을 알려주지 않는다.  
이 앱은 의사를 대신하지 않는다.

AI가 만든 요약은 의사가 참고하는 초안이다.

## 5. 핵심 사용자 흐름

1. 사용자가 앱을 연다.
2. 앱은 AI 사용 안내와 의료진 참고용 안내를 보여준다.
3. 앱은 저장된 기본 프로필과 의료정보가 있는지 확인한다.
4. 필수 정보가 없으면 사용자가 최소 정보만 입력한다.
5. 사용자는 증상을 글, 음성, 사진, 선택 버튼 중 편한 방법으로 입력한다.
6. 앱은 필요한 정보만 MedGemma에 보낸다.
7. 이름, 연락처 같은 직접 식별정보는 MedGemma에 보내지 않는다.
8. MedGemma는 부족한 정보를 확인하고 질문을 하나씩 한다.
9. 체온, 혈압, 혈당 같은 값은 필요할 때만 묻는다.
10. 사용자는 질문에 따라 `예`, `아니오`, `모르겠음` 중 하나를 선택할 수 있다.
11. 사용자는 최대 4~5개 중심 질문에 답한다.
12. 새로 알게 된 의료정보가 있으면 앱이 저장할지 물어본다.
13. 사용자가 허락한 정보만 기본 프로필 또는 의료정보에 저장한다.
14. MedGemma는 의사가 보기 쉬운 S/O 요약 초안을 만든다.
15. 사용자는 병원에서 요약 화면을 의사에게 보여준다.

## 7. S/O 요약이란

이 앱은 의사가 읽기 쉽게 내용을 S와 O로 나눈다.

S는 Subjective의 줄임말이다.  
쉽게 말하면 **사용자가 직접 말한 증상과 느낌**이다.

예:

- 팔꿈치가 아파요.
- 3일 전부터 아팠어요.
- 움직이면 더 아파요.
- 통증은 6점 정도예요.
- 평소와 다르게 어지러워요.

O는 Objective의 줄임말이다.  
하지만 이 앱에서는 더 쉽게 **사용자가 제공한 객관 참고 정보**라고 부른다.

예:

- 키와 몸무게
- 오늘 잰 체온
- 오늘 잰 혈압
- 오늘 잰 혈당
- 첨부한 사진
- 첨부한 영상 파일
- 검사결과 파일
- 복용약
- 알레르기
- 기존 질환
- 가족력

중요한 점:

- MedGemma가 병을 판단하는 것이 아니다.
- MedGemma는 입력된 내용을 정리만 한다.
- 사진을 보고 병명을 말하면 안 된다.
- “염증 같다”, “골절 같다”, “감염 같다” 같은 말은 하면 안 된다.
- 불확실한 내용은 “확인 필요”라고 표시한다.

## 8. AI가 해도 되는 일과 하면 안 되는 일

MedGemma가 해도 되는 일:

- 사용자가 입력한 내용을 정리한다.
- 부족한 정보를 질문한다.
- 저장된 의료정보가 있으면 참고한다.
- 같은 질문을 반복하지 않도록 돕는다.
- S와 O로 내용을 나눈다.
- 의사가 읽기 쉬운 요약 초안을 만든다.
- 저장하면 좋을 정보를 후보로 알려준다.

MedGemma가 하면 안 되는 일:

- 진단명 말하기
- 병명 추측하기
- 치료 방법 추천하기
- 약을 먹으라고 말하기
- 약을 끊으라고 말하기
- 응급인지 확정하기
- 의사를 대신하기
- 사용자 허락 없이 의료정보 저장하기
- 로컬 저장소에 직접 저장하기

## 9. AI 질문 규칙

질문은 기본적으로 4~5개 안에서 끝낸다.

기본 질문 예:

1. 가장 불편한 증상은 무엇인가요?
2. 언제부터 시작됐나요?
3. 어디가, 어느 정도로 불편한가요?
4. 좋아지거나 나빠지는 상황이 있나요?
5. 복용약, 기존 질환, 알레르기 중 알려야 할 것이 있나요?

저장된 의료정보가 있으면 기본값으로 참고한다.

하지만 아래 경우에는 다시 확인할 수 있다.

- 시간이 지나며 바뀔 수 있는 정보다.
- 현재 증상이나 질환과 관련이 크다.
- 변화 과정이 진료에 중요할 수 있다.
- 저장된 날짜가 오래됐다.

예를 들어 복용약, 알레르기, 몸무게, 흡연/음주 여부는 바뀔 수 있다.  
이런 정보는 “저장된 복용약 정보가 지금도 맞나요?”처럼 확인할 수 있다.

기존 질환이나 가족력처럼 잘 바뀌지 않는 정보는 보통 다시 묻지 않는다.  
다만 현재 증상과 관련이 크면 확인할 수 있다.

## 10. 체온, 혈압, 혈당 질문 규칙

체온, 혈압, 혈당은 항상 묻지 않는다.

이 값들은 현재 증상에 도움이 될 때만 물어본다.

예:

- 열이 난다고 하면 체온을 물어볼 수 있다.
- 어지럽다고 하면 혈압이나 혈당을 물어볼 수 있다.
- 숨이 차다고 하면 산소포화도 같은 값이 있는지 물어볼 수 있다.

하지만 사용자가 모를 수 있다.

그래서 바로 값을 요구하지 않는다. 먼저 측정값이 있는지 묻는다.

선택지는 Figma 기준에 맞춘다.

- 예
- 아니오
- 모르겠음

사용자가 `예`를 선택하면 값을 입력한다.  
사용자가 `아니오` 또는 `모르겠음`을 선택하면 값을 입력하지 않고 다음 단계로 간다.

답하지 않아도 문진은 계속 진행된다.  
답하지 않아도 S/O 요약은 만들 수 있다.

입력한 체온, 혈압, 혈당은 그날 문진 기록에만 저장한다.  
기본 프로필이나 의료정보에는 저장하지 않는다.

## 11. 저장되지 않은 의료정보를 물어보는 방식

문진 중 필요한 정보가 기본 프로필이나 의료정보에 없을 수 있다.

예:

- 복용약이 저장되어 있지 않다.
- 알레르기가 저장되어 있지 않다.
- 가족력이 저장되어 있지 않다.
- 기존 질환이 저장되어 있지 않다.

이때 AI는 질문할 수 있다.

예:

```text
복용 중인 약이 있나요?
저장된 정보가 없어서 한 번만 확인할게요.
```

사용자가 답하면 앱은 이렇게 물어본다.

```text
이 정보를 의료정보에 저장할까요?
[저장하기] [이번 문진에만 사용]
```

AI가 직접 저장하지 않는다.  
앱이 사용자에게 확인한다.  
사용자가 허락해야 저장한다.

저장 후보 예:

```json
{
  "nextQuestion": "복용 중인 약이 있나요?",
  "saveCandidate": {
    "category": "medications",
    "value": "혈압약",
    "savePolicy": "confirm",
    "reason": "다음 문진에서 다시 묻지 않기 위해"
  }
}
```

## 12. Hugging Face와 MedGemma 연결

사용할 모델:

```text
google/medgemma-1.5-4b-it
```

이 모델을 쓰는 이유:

- 4B 모델이라 비교적 가볍다.
- 글과 이미지를 함께 받을 수 있다.
- MedGemma 1.5 계열이다.
- 무료 범위 데모에 가장 현실적이다.

연결 방식:

1. 사용자가 앱에서 문진을 입력한다.
2. Next.js 서버 API가 요청을 받는다.
3. 서버 API가 Hugging Face에 요청을 보낸다.
4. Hugging Face가 MedGemma 결과를 돌려준다.
5. 앱이 결과를 화면에 보여준다.

중요한 보안 규칙:

- Hugging Face 토큰은 브라우저에 넣지 않는다.
- 토큰은 서버의 `.env` 파일에만 둔다.
- 사용자의 이름과 연락처는 AI에 보내지 않는다.

무료 범위에서만 데모한다.

무료 사용량이 끝나거나 연결이 실패하면 이렇게 보여준다.

```text
AI 응답을 잠시 사용할 수 없어요.
잠시 후 다시 시도해 주세요.
```

사용자는 다시 시도할 수 있어야 한다.  
AI가 안 되어도 수동 요약 화면을 볼 수 있어야 한다.

## 13. 사진과 영상 처리

### 사진

- 사용자는 사진을 첨부할 수 있다.
- 사진은 필요할 때 MedGemma에 보낼 수 있다.
- 사진은 저장하기 전에 앱에서 크기를 줄인다.
- 사진은 가로 또는 세로 길이가 최대 1024px 정도가 되도록 줄인다.
- 사진은 JPEG 같은 가벼운 형식으로 바꿔 저장한다.
- 이렇게 해야 사용자 기기의 저장 공간을 너무 많이 쓰지 않는다.
- AI는 사진을 보고 병명을 말하면 안 된다.
- AI는 사진에 대해 확실하지 않은 해석을 하면 안 된다.
- 사진은 의사가 직접 확인할 수 있도록 첨부로 보여준다.

### 영상

- 영상은 분석하지 않는다.
- 영상은 첨부 파일로만 저장한다.
- 요약에는 파일명과 첨부 여부만 보여준다.
- 의사가 필요하면 직접 확인한다.

## 14. 음성 입력

브라우저 음성 인식을 우선 사용한다.

필요한 기능:

- 말하기 시작 상태 표시
- 앱이 듣고 있다는 것을 보여주는 오디오 웨이브 또는 비슷한 움직임
- 말하기 종료 상태 표시
- 인식된 문장을 사용자가 고칠 수 있음
- 음성 인식 실패 시 글 입력으로 바꿀 수 있음
- 사용자가 확인하기 전에는 자동 제출하지 않음

말하기 종료 규칙:

- 사용자가 말하다가 잠시 멈출 수 있다.
- 그래서 너무 빨리 말하기를 끝내면 안 된다.
- 음성 인식 중 3.5초 이상 말소리가 감지되지 않으면 말하기가 끝난 것으로 본다.
- 그 뒤 텍스트 변환 단계로 넘어간다.
- 사용자는 변환된 문장을 확인하고 고칠 수 있어야 한다.

## 15. 저장 방식

별도 데이터베이스는 만들지 않는다.

사용자 기기 안에 저장한다.

저장 위치:

- 기본 저장: IndexedDB
- 간단한 설정: localStorage 사용 가능

사진 저장 규칙:

- 사진은 IndexedDB에 저장하기 전에 반드시 압축한다.
- 사진의 긴 쪽 길이는 최대 1024px 정도로 줄인다.
- 압축된 사진만 저장한다.
- 원본 사진을 그대로 저장하지 않는다.
- 저장 공간이 부족하면 사용자에게 알려준다.

저장하는 것:

- 기본 프로필
- 의료정보
- 문진 기록
- 첨부 파일 정보
- AI 사용 동의 여부
- 잠금 설정
- 저장 후보 정보

기본 프로필에 저장하는 것:

- 생년월일 또는 나이
- 성별

의료정보에 저장하는 것:

- 키
- 몸무게
- 기존 질환
- 복용약
- 알레르기
- 가족력
- 수술/시술 이력
- 흡연/음주 여부

문진 기록에만 저장하는 것:

- 오늘 증상
- 통증 강도
- 시작 시점
- 증상 변화
- 오늘 잰 체온
- 오늘 잰 혈압
- 오늘 잰 혈당
- 사진 첨부
- 영상 첨부

삭제 기능:

- 모든 기록 삭제
- 기본 프로필 삭제
- 의료정보 삭제
- 특정 문진 삭제

## 16. 개인정보와 동의

이름, 연락처, 주민등록번호 같은 정보는 AI에 보내지 않는다.

AI에 보낼 수 있는 정보:

- 나이
- 성별
- 키/몸무게
- 기존 질환
- 복용약
- 알레르기
- 가족력
- 현재 증상
- 첨부 사진

처음 사용할 때 아래 안내를 보여준다.

```text
AI 요약 생성을 위해 입력한 증상 정보가 외부 AI 서비스로 전송될 수 있어요.
이름이나 연락처 같은 정보는 보내지 않아요.
```

## 17. 응급 신호 처리

앱은 응급인지 진단하지 않는다.

하지만 위험할 수 있는 말이 입력되면 안전 안내를 보여준다.

예:

- 가슴 통증과 숨참
- 갑자기 한쪽 팔이나 다리에 힘이 빠짐
- 의식이 흐림
- 심한 호흡곤란
- 갑자기 심한 두통
- 피가 많이 남

안내 예:

```text
응급일 수 있어요.
증상이 심하거나 계속되면 즉시 주변 사람이나 의료진에게 알려주세요.
```

이 안내는 진단이 아니다.  
안내가 떠도 문진과 요약 생성은 계속할 수 있다.

## 18. 접근성 원칙

이 앱은 쉽게 써야 한다.

그래서 아래 원칙을 지킨다.

- 버튼은 최소 48px 이상으로 만든다.
- 본문 글자는 최소 16px 이상으로 만든다.
- 중요한 버튼은 누르기 쉽게 만든다.
- 질문은 한 번에 하나만 보여준다.
- 어려운 의료 용어를 줄인다.
- 아이콘만 쓰지 않고 글자도 함께 보여준다.
- 색깔만으로 의미를 전달하지 않는다.
- 오류, 로딩, 완료 상태를 글자로 알려준다.
- 글자 크기를 키워도 화면이 깨지지 않게 한다.
- 사용자가 확인하기 전에는 자동 제출하지 않는다.

### 첨부 실패

처리:

- 파일 크기나 파일 형식을 알려준다.
- 다시 첨부할 수 있게 한다.

### 저장 실패

## 20. 테스트 기준

### 기능 테스트

- Figma 기준 주요 화면이 있다.
- 아래쪽 탭, 헤더, 버튼, 입력창이 Figma와 비슷하다.
- 기본 프로필을 입력하면 저장된다.

- 저장된 정보는 다음 문진에서 다시 사용된다.

- 음성 입력 결과를 고칠 수 있다.
- 음성 입력 중 앱이 듣고 있다는 표시가 보인다.
- 3.5초 이상 말소리가 없으면 말하기 종료로 처리된다.
- 사진을 첨부할 수 있다.
- 사진은 저장 전에 최대 1024px 정도로 압축된다.
- 영상은 첨부 목록에만 보인다.

- AI 실패 시 수동 요약을 볼 수 있다.
- 의료진용 S/O 요약이 만들어진다.

### 안전 테스트

- AI가 진단명을 만들지 않는다.
- AI가 치료 방법을 추천하지 않는다.
- AI가 약 복용을 지시하지 않는다.
- 응급 신호는 진단이 아니라 안전 안내만 보여준다.
- 이름과 연락처는 AI 요청에 포함되지 않는다.

### 접근성 테스트

- 모바일 화면에서 버튼 크기가 최소 48px 이상이다.
- 본문 글자 크기가 최소 16px 이상이다.
- 글자를 키워도 내용이 잘리지 않는다.
- 색깔 없이도 상태를 이해할 수 있다.
- 화면 읽기 순서가 자연스럽다.

### 디자인 비교 테스트

- Figma `최종 prototype`과 실제 화면을 비교한다.
- 393px 모바일 폭에서 Figma와 비슷하게 보인다.
- `Buttons`, `Chat_Input`, `speech_bubble`, `Summary_Item`, `FormInput`, `GNB`, `Header` 역할을 따른다.
- Figma에 없는 설명 화면을 마음대로 많이 추가하지 않는다.
